# Prompt: Tech Radar — Software, AI & Homelab

## Objetivo

Generar un newsletter técnico, fácil de leer y útil para developers full-stack e IT.

El contenido debe cubrir tendencias de ingeniería de software, con énfasis especial en frontend moderno y orquestación de IA aplicada a productos reales.

## Formato de salida

Crear dos versiones:

1. Texto plano de respaldo.
2. HTML mobile-first compatible con clientes de correo.

## Prioridad editorial

Cada edición debe sentirse fuerte en dos ejes:

1. Frontend moderno y web platform.
2. Orquestación de IA.

Reglas concretas:

- Incluir al menos 2 temas de frontend, JavaScript, TypeScript, CSS, diseño UI, accesibilidad o web platform.
- Incluir al menos 2 temas de IA aplicada u orquestación de IA.
- Seleccionar entre 7 y 8 temas reales.
- Mantener diversidad de fuentes y vendors.

## Qué cuenta como orquestación de IA

La orquestación de IA incluye:

- Agentes y agent workflows.
- Tool calling.
- MCP y conexión con herramientas externas.
- RAG pipelines.
- Evals y testing de sistemas con IA.
- Tracing, observabilidad y debugging de agentes.
- Routing de modelos.
- Memory y context engineering.
- Human-in-the-loop.
- Timeouts, retries, colas y control de costos.
- Frameworks como LangGraph, LangChain, LlamaIndex, Vercel AI SDK, OpenAI Agents/Responses, HumanLayer, LiteLLM o vLLM.

## Reglas de diversidad

- Usar entre 7 y 8 temas reales.
- Cubrir al menos 5 áreas distintas.
- Evitar que una sola empresa, framework o dominio domine la edición.
- Máximo 1 tema por vendor principal, salvo excepción justificada.
- Usar fuentes originales o cercanas a la fuente original.

## Distribución recomendada

- 2 tarjetas frontend/web platform.
- 2 tarjetas IA/orquestación de IA.
- 1 tarjeta backend/arquitectura.
- 1 tarjeta DevOps/platform/observabilidad.
- 1 tarjeta homelab o repo radar.
- 1 tarjeta flexible según relevancia.

## Áreas sugeridas

1. Frontend, JavaScript o TypeScript.
2. CSS, diseño UI, UX engineering, accesibilidad o web platform.
3. Backend, runtime, bases de datos o arquitectura de software.
4. DevOps, platform engineering, CI/CD, testing u observabilidad.
5. IA aplicada a desarrollo, agentes, modelos, tooling o evaluación.
6. Orquestación de IA: agent workflows, tool calling, MCP, RAG, evals, tracing, routing, memory/context engineering o human-in-the-loop.
7. Homelab, self-hosting, Linux, networking, contenedores o automatización personal.
8. Arquitectura de software: distributed systems, event-driven, resiliencia, performance, API design o cloud architecture.
9. Repo Radar: repositorio open source relevante con release o actividad reciente.

## Secciones obligatorias

- Resumen editorial.
- Esta edición, en breve.
- 7 a 8 tarjetas de temas.
- Qué vale la pena probar esta semana.
- Lectura rápida.
- Fuentes de esta edición.

## Validación final

- No hay placeholders visibles.
- Todos los links apuntan a fuentes originales.
- Hay 7 u 8 tarjetas.
- Hay al menos 2 temas frontend/web platform.
- Hay al menos 2 temas de IA/orquestación de IA.
- Hay al menos 5 áreas cubiertas.
- Ningún vendor domina la edición.
- Existe una versión de texto plano y una versión HTML.
